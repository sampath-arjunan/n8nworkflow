# AI sticky generator for n8n templates

### 1. Workflow Overview

This workflow, titled **AI sticky generator for n8n templates**, is designed to automatically generate a concise, structured sticky note overview for any given n8n workflow template. It accepts a complete workflow JSON as input, then uses an AI agent powered by OpenAI’s language model to analyze the workflow structure and produce a single main sticky note summarizing the workflow’s purpose, operation steps, and setup instructions.

The workflow logic is organized into three main functional blocks:

- **1.1 Input Reception:** Receives and prepares the full workflow JSON input.
- **1.2 AI Processing:** Sends the JSON input to an AI language model agent configured with a prompt to generate a sticky note overview.
- **1.3 Output Handling:** Parses and formats the AI response into a sticky note node within the workflow.

### 2. Block-by-Block Analysis

#### 1.1 Input Reception

**Overview:**  
This block provides the workflow JSON input to the AI agent. It includes a manual trigger to start the workflow and a node to set the JSON data that represents the workflow to be summarized.

**Nodes Involved:**  
- Execute workflow  
- Paste workflow JSON here

**Node Details:**  

- **Execute workflow**  
  - Type: Manual Trigger  
  - Role: Starts the workflow execution manually.  
  - Configuration: No parameters; simply triggers downstream nodes.  
  - Inputs: None (trigger node)  
  - Outputs: Connected to “Paste workflow JSON here”  
  - Failure modes: None typical, but manual trigger might be forgotten leading to no runs.

- **Paste workflow JSON here**  
  - Type: Set  
  - Role: Holds the JSON object of the workflow to be analyzed.  
  - Configuration: A single field `json` of type Object expected to contain the full workflow JSON. Requires the user to paste or provide JSON here before running.  
  - Inputs: Receives trigger from “Execute workflow”  
  - Outputs: Connected to “AI Agent” node  
  - Failure modes: If JSON is empty or malformed, AI agent may fail or produce invalid output. No validation here.

---

#### 1.2 AI Processing

**Overview:**  
This block uses an AI language model agent to analyze the input workflow JSON and generate a markdown text for the main sticky note that summarizes the workflow’s purpose, how it works, and setup instructions.

**Nodes Involved:**  
- AI Agent  
- OpenAI Chat Model  
- Structured Output Parser1

**Node Details:**  

- **AI Agent**  
  - Type: Langchain Agent (n8n-nodes-langchain.agent)  
  - Role: Orchestrates the AI prompt and output processing using the configured language model and output parser.  
  - Configuration:  
    - Input text expression embeds the workflow JSON from “Paste workflow JSON here” node as a string.  
    - System message prompt instructs the AI to generate a single markdown sticky note overview with strict formatting and content rules.  
  - Inputs: Receives JSON string from “Paste workflow JSON here”  
  - Outputs: Sends request to “OpenAI Chat Model” and receives parsed response from “Structured Output Parser1”  
  - Failure modes:  
    - Large JSON input could exceed model token limits or timeout.  
    - Expression failures if JSON path is incorrect.  
    - AI model might produce invalid JSON if prompt misunderstood.  

- **OpenAI Chat Model**  
  - Type: Langchain LM Chat OpenAI  
  - Role: Provides GPT-5-mini model interaction for natural language generation.  
  - Configuration: Uses “gpt-5-mini” model with credentials linked to OpenAI account.  
  - Inputs: Receives prompt from “AI Agent” node.  
  - Outputs: AI-generated markdown response to “Structured Output Parser1”  
  - Failure modes: API key expiration, rate limits, or network issues.  

- **Structured Output Parser1**  
  - Type: Langchain Output Parser (Structured)  
  - Role: Validates and structures AI output into the expected JSON format for a sticky note node.  
  - Configuration: Provides a JSON schema example for a sticky note node containing markdown content.  
  - Inputs: Receives raw AI output from “OpenAI Chat Model”  
  - Outputs: Structured JSON for sticky note node back to “AI Agent”  
  - Failure modes: If AI output does not match schema, parsing will fail.

---

#### 1.3 Output Handling

**Overview:**  
This block is implicit within the AI Agent’s output handling; the final output is the generated sticky note JSON, which can be used to create or update a sticky note node in n8n.

**Nodes Involved:**  
- None explicitly; handled inside the AI Agent node’s output.

**Node Details:**  
- The AI Agent outputs a single JSON object defining a sticky note node with a height of 750 and width of 500, containing the markdown overview generated by the AI.  
- This JSON object can be used by the workflow user to insert a sticky note summarizing the workflow template.

### 4. Reproducing the Workflow from Scratch

1. **Create a Manual Trigger node:**  
   - Name: Execute workflow  
   - Purpose: To manually start the workflow.

2. **Add a Set node:**  
   - Name: Paste workflow JSON here  
   - Connect the Manual Trigger node to this node.  
   - Configure one field:  
     - Field name: `json`  
     - Type: Object  
     - Value: Leave empty initially; this is where the user pastes the full workflow JSON to summarize.

3. **Add a Langchain Agent node:**  
   - Name: AI Agent  
   - Connect the “Paste workflow JSON here” node to this node.  
   - Parameters:  
     - Set the `text` parameter to an expression that converts the JSON from the previous node to a string, e.g., `=Here is the workflow JSON: {{ $('Paste workflow JSON here').item.json.json.toJsonString() }}`  
     - In the options, create a system message prompt instructing the AI to generate a single markdown sticky note with a main overview, including a title, “How it works” steps, and “Setup” checklist, all within 100–300 words, following the detailed formatting and content rules.

4. **Add a Langchain LM Chat OpenAI node:**  
   - Name: OpenAI Chat Model  
   - Connect the AI Agent node’s language model output to this node.  
   - Select model “gpt-5-mini” (or an equivalent GPT model).  
   - Configure using OpenAI API credentials (set up your OpenAI account in n8n).  
   - Keep default options.

5. **Add a Langchain Output Parser Structured node:**  
   - Name: Structured Output Parser1  
   - Connect the OpenAI Chat Model’s output to this node.  
   - Provide a JSON schema example for a sticky note node (including fields: `content` with the markdown string, `height` 750, `width` 500).  
   - This parser ensures AI output conforms to the expected sticky note JSON format.

6. **Connect the parser output back to the AI Agent’s output parser input.**

7. **Save and activate the workflow.**

8. **Usage:**  
   - Paste the target workflow JSON into the “Paste workflow JSON here” node’s `json` field.  
   - Trigger the workflow manually via the “Execute workflow” node.  
   - The AI Agent will generate a single sticky note JSON with the overview markdown, which can be used to create a sticky note in the editor or exported.
